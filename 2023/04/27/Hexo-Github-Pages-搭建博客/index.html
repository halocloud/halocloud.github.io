Hexo + Github Pages 博客搭建指南

本文最后更新于：2023年5月9日 晚上

前言

  • Hexo

Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

  • GitHub Pages

GitHub Pages可以被认为是用户编写的、托管在github上的静态网页。使用GitHub Pages可以为你提供一个免费的服务器，免去了自己搭建服务器和写数据库的麻烦。此外还可以绑定自己的域名。

环境准备

  • Node.js
  • Git

在路径E:\HelloWorld\HaloBlog\Hexo 右键 Git Bash Here

Hexo 部署

  1. 准备GitHub账户：这里注册账户为 halocloud
  2. 创建项目代码库：项目名必须为用户名.github.io 这里为 halocloud.github.io
  3. 配置 SSH 密钥：由于个人原因，我本机电脑上有两个GitHub账号，所以这里配置密钥需要如下处理

原理分析：

  • SSH的公钥是GitHub作为本地仓库和远程仓库连接的唯一标识，一个公钥只能对应一个GitHub账户，如果将一个相同的公钥上传到不同的GitHub账户，GitHub则无法做出辨识，进而导致错误
  • 一台电脑，可以生成多对公私钥，可以通过配置，将不同的公钥上传到不同的GitHub账号，那么就不存在单个公钥绑定多个GitHub账号的情况存在了

查看当前密钥
打开终端输入ls ~/.ssh/可以查看当前已有的密钥，显示id_rsa与 id_rsa_pub说明已经有一对密钥
创建新的密钥
首先使用以下命令进入 SSH根目录下cd ~/.ssh/
使用以下命令创建新密钥 ssh-keygen -t rsa -f ~/.ssh/这里是新密钥名称 -C "这里是你的邮箱"
我这里使用ssh-keygen -t rsa -f ~/.ssh/halo -C "xxxxxx@gmail.com"
配置config
查看.ssh/根路径下, 有没有config文件，没有则新建一个。
内容为：

#第一个账号，默认使用的账号，不用做任何更改
Host github.com
HostName github.com
User git
IdentityFile ~/.ssh/id_rsa

#第二个账号
Host halo.github.com
HostName github.com
User git
IdentityFile ~/.ssh/halo
注意：halo.github.com 后续会用到
设置新GitHub账户SSH key
在.ssh目录下找到你创建的新的公钥，文件名为新密钥的名称.pub，比如我的是halo.pub，用记事本打开，复制里面的内容，然后打开你的新GitHub账号主页，依次进入Settings —> SSH and GPG keys —> New SSH key，将刚复制的内容粘贴到Key那里，Title可以随便填，点击Add Key保存。
清空本地的 SSH 缓存，添加新的 SSH 密钥 到 SSH agent中

使用命令cd ~/.sshcd到.ssh根目录下，依次执行以下命令：

验证配置是否成功
依次执行以下命令，第一个为默认ssh_key验证；第二个为新的ssh_key验证，其中“xxxxxx”为你先前在config文件中的命名
ssh -T git@github.com
ssh -T git@halo.github.com
显示以下信息说明配置成功:Hi 你的用户名! You've successfully authenticated, but GitHub does **not** provide shell **access**.
单独设置用户名/邮箱
进入Hexo博客.git目录下执行以下命令单独设置用户名/邮箱：
git config user.name "halocloud"
git config user.email "xxxxxx@gmail.com"
如果此时报错：fatal: not in a git directory，说明你没有进入.git目录下，具体路径：\Hexo.deploy_git.git，.git目录是隐藏的，需要你设置隐藏目录可见
执行以下命令可以查看设置是否成功 git config *--list*

  1. 安装部署命令
  1. 配置 _config.yml deploy信息

deploy:
type: git
repo: git@halo.github.com:halocloud/halocloud.github.io.git
branch: master

注意这里使用了上面的 halo.github.com

  1. 编译部署

这里就部署成功，登录GitHub上查看 halocloud.github.io 项目，此时文件已经提交到master。

访问 https://halocloud.github.io/ 就可以看到效果了。
